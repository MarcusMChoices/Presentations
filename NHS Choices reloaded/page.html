Introduction

Talk on 3 subjects:

  • New NHS Choices home page and site refresh
  • HTML5 and CSS3
  • Three day course — 'HTML5 and Beyond'

NHS Choices site refresh

  • Current home page designed in 2010 (release 15!), Health A–Z & Live well sections even earlier (2009)
  • Political reasons
  • Site refresh based on user research — what the users want
    • 8 rounds of user testing so far (more details on our staff site)
    • Looked at branding, values, imagery, styling, tone, layouts, content
    • Various design routes were created and retested
  • Technology constraints
    • SharePoint
    • Brightcove video players
  • Has resulted in
    • New home page
    • Live well hub, Live well article
    • Health AZ hub, Health AZ condition article
    • Health news hub, Health news article

CSS3/HTML5

  • Opportunity to use current browser features (that have been around for a while)
  • Able to do so because IE6 support has been dropped
  • Still have to support IE7/8, so use of new HTML5 tags e.g. <header>, <nav>, <footer> limited
    • Could use 'polyfills' or 'shims/shivs', which is usually javascript to provide the functionality that the browser should provide natively, e.g. Modernizr, html5shiv
    • Extra js call
    • Changes to template markup
    • Time constraints

Newer CSS features used

  • Border radius
  • Box shadow
  • Text shadow
  • Pseudo classes (such as :first-child, :checked)

Efficiency

Implemented on the site now:

  • Base64 encoding images
    • Reduces server calls
    • Load instantly with the css
  • Use of pseudo selectors — :first-child
  • Text shadow
  • Caching — Akamai, CDN

Future nice to haves:

  • Minification of CSS and JS and HTML
  • GZip components
  • Inline some CSS and JS — above the fold content

Reduce downloaded resources — HTTP requests are expensive!
